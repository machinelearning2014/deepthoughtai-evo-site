Core Principles

Prolog-first

Derivation-based

Explicit Assumptions

Consistency Verification

Tool Discipline

EVO Text Description

EVO (Explicit-assumption Verification Orchestrator) is a Prolog-first autonomous reasoning system built for traceable, consistent, and auditable conclusions. Instead of treating large language models as primary reasoners, EVO formalizes each task as facts, rules, assumptions, and constraints inside a Prolog knowledge base.

Reasoning is accepted only when a target conclusion is derivable through Prolog inference with a proof trace. If required information is missing, EVO can call external tools, but those outputs must be mapped back into explicit Prolog facts before any new inference is attempted.

A key design choice is explicit assumptions. Assumptions are not hidden bridges: they are represented as first-class objects that can be toggled and stress-tested. EVO checks whether key conclusions still hold when assumptions are removed, allowing users to see robustness and dependency directly.

Before answering, EVO performs consistency verification. If integrity constraints are violated, the system halts or repairs the knowledge base rather than returning an ungrounded response. This creates a stronger guarantee of logical coherence than purely neural pipelines.

The architecture combines a Prolog reasoning core, a controlled tool interface, optional domain verifiers, and a user-facing presentation layer for conclusions, proofs, and assumption impacts. This pattern supports domain overlays such as mathematics and legal support while preserving the same core derivation discipline.
